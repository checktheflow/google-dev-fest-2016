HTTP/2

  • Multiplexed Streams: connection is multiplexed
  • single connection requires just one handshake (SSL/TLS)
  • no more domain sharding, concatenation, image spriting
  • Server Push: proactive sending assets (like JS and CSS)
  • Header Compression: all the headers are packed into one compressed block to be sent as a unity
  • Binary Instead of Textual Format: less code, easier parsing on the server
  • Exampe: A grid of 180 tiled images

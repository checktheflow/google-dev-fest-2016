Below the Fold / Critical Path

  • Bevor der Browser die Seite rendern kann, muss er den DOM und das CSSOM aufbauen.
  • Sowohl das DOM als auch das CSSOM lassen sich durch JS verändern.
  • Daher sollten nicht benötigte Scripte und Stylesheets asynchron geladen werden.

Below the Fold / Critical Path

  • Nur das ausliefern, was initial benötigt wird, den Rest später nachladen!

HTTP/2

  • Multiplexed Streams: Die Verbindung wird für verschiedene Inhalte genutzt
  • Eine einzelne Verbindung benötigt nur ein Handshake(SSL/TLS)
  • Domain Shardiding, Concatenation, Image Spriting etc. wird nicht mehr benötigt
  • Server Push: Proaktiv Assets (wie JS and CSS) an den Browser senden
  • Header Compression: Alle Header weren in einem Block als komprimierte Einheit verschickt
  • Binary anstatt textuelles Format: weniger Code, leichteres Parsing auf dem Server
  • Beispiel: A grid of 180 tiled images

Single Page Applications

  • Initialier Download oft schnell aber Rendering erst nach den ersten AJAX-Requests
  • Pre-Rendering auf dem Server
  • Skeletons
  • ...

Laufzeit­performance (60 fps)

  • Die meisten Geräte rendern mit 60fps
  • Zeit für Berechnungen sind dann nur ca. 10ms
